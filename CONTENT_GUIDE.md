# Инструкция по редактированию контента сайта ОстеоКлиник

## Общие сведения

Сайт **osteo-klinik.by** — это статический сайт, собранный на фреймворке Astro. Весь контент хранится в виде файлов в репозитории на GitHub. Чтобы изменить что-то на сайте, нужно:

1. Отредактировать нужный файл в репозитории
2. Сохранить изменение (сделать «коммит»)
3. Сайт автоматически пересоберётся и обновится через 2–3 минуты

Все правки можно делать прямо в браузере через интерфейс GitHub — устанавливать программы не нужно.

---

## Как редактировать файлы на GitHub

1. Откройте репозиторий проекта на GitHub
2. Перейдите к нужному файлу, кликая по папкам
3. Нажмите иконку карандаша (Edit this file) в правом верхнем углу файла
4. Внесите изменения прямо в редакторе
5. Нажмите зелёную кнопку **"Commit changes..."**
6. В появившемся окне оставьте краткое описание (например: «Обновлён телефон в шапке») и нажмите **"Commit changes"**
7. Через 2–3 минуты изменения появятся на сайте

## Что и где менять

### 1. Услуги (описания направлений)

**Где:** папка [`src/content/services/`](src/content/services/)

Каждая услуга — отдельный файл `.md`:

- `neurology.md` — Неврология
- `ultrasound.md` — УЗИ диагностика
- `surgery.md` — Хирургия
- `cardiology.md` — Кардиология
- `phlebology.md` — Флебология

**Структура файла:**

Файл делится на две части — «шапку» (между `---`) и основной текст.

```
---
title: "Неврология"
subtitle: "Диагностика и лечение заболеваний нервной системы"
slug: "neurology"
cardText: "Текст для карточки на главной странице"
order: 1
symptoms:
  - "Головные боли и мигрени"
  - "Боли в спине и шее"
methods:
  - "Осмотр и консультация невролога"
  - "Медикаментозная терапия"
---

Здесь идёт основной текст страницы. Можно использовать:
## Заголовки (с ##)
- Списки (с -)
Обычный текст абзацами.
```

**Правила:**

- Не менять поле `slug` — от него зависит адрес страницы
- Поле `order` — порядок отображения на главной (1 = первый)
- Списки `symptoms` и `methods` — каждый пункт с новой строки, начинается с `  - "текст"`
- Все значения в шапке — в кавычках `"..."`

### 2. Врачи

**Где:** папка [`src/content/doctors/`](src/content/doctors/)

Файлы: `ivanov.md`, `petrova.md`, `kuznetsova.md`, `volkov.md`, `sidorov.md`

**Структура файла:**

```
---
name: "Иванов Сергей Петрович"
role: "Невролог"
experience: "18 лет"
tags: ["неврология", "остеохондроз", "головные боли"]
order: 1
---

Текст-описание врача. Специализация, квалификация и т.д.
```

**Правила:**

- `tags` — ключевые слова через запятую в квадратных скобках, каждое в кавычках
- `order` — порядок отображения на странице «Врачи»

**Чтобы добавить нового врача:**

- Создать новый файл в папке `src/content/doctors/` (например, `novikov.md`)
- Заполнить по образцу любого существующего файла
- На GitHub: в папке `doctors` нажать **"Add file" → "Create new file"**, вписать имя файла с `.md` и заполнить содержимое

**Чтобы удалить врача:**

- Удалить его файл из папки `src/content/doctors/`

### 3. Блог (статьи)

**Где:** папка [`src/content/blog/`](src/content/blog/)

**Структура файла:**

```
---
title: "Заголовок статьи"
date: 2026-03-28
description: "Краткое описание для карточки в списке статей."
---

Текст статьи. Можно использовать:
## Подзаголовки
- Списки
**Жирный текст** и *курсив*
```

**Правила:**

- `date` — дата в формате `ГГГГ-ММ-ДД` (год-месяц-день)
- Имя файла станет частью URL. Используйте латиницу и дефисы, например: `new-article-name.md`
- Чтобы добавить статью — создайте новый `.md` файл в папке `blog`

### 4. Цены

**Где:** файл [`src/data/prices.json`](src/data/prices.json)

Это файл в формате JSON. Структура:

```json
[
  {
    "category": "Неврология",
    "items": [
      { "name": "Первичная консультация невролога", "price": "50 BYN" },
      { "name": "Повторная консультация невролога", "price": "40 BYN" }
    ]
  },
  {
    "category": "Следующая категория",
    "items": [...]
  }
]
```

**Правила — очень важно соблюдать формат:**

- Каждая строка с услугой: `{ "name": "Название", "price": "XX BYN" }`
- Между элементами списка — запятые, но после последнего элемента в списке запятой быть **не** должно
- Все тексты — в двойных кавычках `"..."`
- Если нужно добавить новую услугу — добавьте строку по образцу в нужную категорию
- Если нужна новая категория — добавьте блок `{ "category": "...", "items": [...] }` через запятую

**Частая ошибка:** лишняя или пропущенная запятая ломает весь файл. Будьте внимательны.

### 5. Телефон, адрес, email

Контактная информация повторяется в нескольких файлах. При изменении нужно обновить **все** места:

| Что | Файл | Что искать |
| --- | --- | --- |
| Телефон в шапке сайта | [`src/components/Header.astro`](src/components/Header.astro) | `+375 29 805-02-02` и `tel:+375298050202` |
| Телефон и адрес в подвале | [`src/components/Footer.astro`](src/components/Footer.astro) | `+375 29 805-02-02`, адрес, email |
| Телефон на главной (кнопка «Позвонить» и CTA) | [`src/pages/index.astro`](src/pages/index.astro) | `tel:+375298050202` и `+375 29 805-02-02` |
| Адрес, часы работы на странице «Контакты» | [`src/pages/contacts.astro`](src/pages/contacts.astro) | Адрес и телефон |

**Важно:** ссылка `tel:+375298050202` (без пробелов) — это техническая ссылка для звонка с мобильного. Текст `+375 29 805-02-02` — это то, что видит пользователь. Менять нужно оба значения.

### 6. Текст на главной странице

**Где:** [`src/pages/index.astro`](src/pages/index.astro)

Здесь находятся:

- **Слайды карусели** (4 штуки) — заголовки, описания, кнопки
- **Полоса преимуществ** («Опытные врачи», «Современное УЗИ» и т.д.)
- **Секция «Направления»** — заголовок и описание (карточки подтягиваются из файлов услуг автоматически)
- **CTA-блок** («Нужна консультация врача...»)

Текст можно менять напрямую — он написан на русском и легко находится в файле.

### 7. Картинки

**Где:** папка [`public/images/`](public/images/)

- `public/images/hero/slide-1.webp` … `slide-4.webp` — фоновые изображения карусели на главной
- `public/images/logo.svg`, `logo-mark.svg` — логотип

**Чтобы заменить картинку:**

1. Подготовьте новое изображение в формате `.webp` (для слайдов) или `.svg` (для логотипа)
2. Назовите файл **точно так же**, как старый (например, `slide-1.webp`)
3. На GitHub: перейдите в нужную папку, нажмите **"Add file" → "Upload files"**, загрузите файл с тем же именем — он перезапишет старый

**Рекомендации для слайдов:** ширина не менее 1920 пикселей, формат `.webp`, размер файла желательно до 200–300 КБ.

## Как добавить новую услугу (полный цикл)

1. Создайте файл в `src/content/services/` (например, `therapy.md`) — по образцу любого существующего
2. Укажите уникальный `slug` (например: `therapy`)
3. Добавьте цены в `src/data/prices.json` — новую категорию или пункты в существующую
4. Если нужна ссылка в меню — добавьте строку в [`src/components/Header.astro`](src/components/Header.astro) и [`src/components/Footer.astro`](src/components/Footer.astro)

## Публикация изменений

После каждого сохранения (коммита) в ветку `main` автоматически запускается сборка сайта через GitHub Actions. Процесс занимает 2–3 минуты. Статус можно проверить на вкладке **Actions** в репозитории на GitHub.

Если сборка завершилась с ошибкой (красный крестик) — скорее всего, допущена ошибка в формате файла (например, пропущена кавычка или запятая в JSON). В этом случае изменения **не** опубликуются, и на сайте останется предыдущая версия.

## Краткая шпаргалка по Markdown

В файлах `.md` используется простая разметка:

- `## Заголовок` — заголовок второго уровня
- `### Подзаголовок` — заголовок третьего уровня
- `- пункт` — элемент списка
- `**жирный текст**` — жирный текст
- `*курсив*` — курсив
- Пустая строка — разделяет абзацы
